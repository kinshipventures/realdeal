Principles
Empty State Rules
Use the orb language
Every empty state illustration uses a colored orb with a line icon inside. The orb color matches the context (pod color, brand green, or neutral). Consistent with the visual system.
Ghost hints show what's coming
Dashed outline "ghost" cards show the shape of the data that will appear. The user can see the structure before it's filled. Reduces cognitive load on first use.
One clear action
Every empty state has exactly one primary green button. No choice paralysis. If there's a secondary action, it's visually recessed. The user always knows what to do next.
