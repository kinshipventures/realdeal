Side-by-side: how every surface, token, and component adapts from light to dark.

Token Mapping
Light to Dark Adaptation
Not just "invert everything." Surfaces go dark, brand green shifts slightly for contrast, text flips to white hierarchy, orb glows get more prominent on dark canvas.
TokenLightDarkNotes
--color-bg #f8f8f6 → #111111 Pure dark, not gray. Lets orb glows pop.
--color-surface #ffffff → #1a1a1a Cards, panels. One step up from bg.
--surface-panel rgba(255,255,255,0.92) → rgba(255,255,255,0.06) Frosted overlay stays translucent on both.
--color-brand #25B439 → #2ECB45 Slightly brighter for contrast on dark surfaces.
--header-band-bg #25B439 → #1B8A2A Darker green band. Less bright on dark bg.
--text-primary rgba(0,0,0,0.82) → rgba(255,255,255,0.87) Not pure white — 87% avoids eye strain.
--text-secondary rgba(0,0,0,0.45) → rgba(255,255,255,0.45) Same opacity, flipped base.
--border rgba(0,0,0,0.07) → rgba(255,255,255,0.06) Subtle edge definition.
--orb-glow opacity 0.25 → opacity 0.40 Glows intensify on dark canvas. More atmosphere.
--hub-orb #1C1C1E → #2C2C30 + ring Slightly lighter + subtle edge ring so it doesn't vanish.
accent tags reach out → reach out Brighter green text, slightly more opaque bg.

Principles
Dark Mode Rules
Orb glows intensify
Glow opacity goes from 0.25 to 0.40 on dark canvas. The orb map becomes the visual hero in dark mode — colors pop against black like a constellation.
Green band darkens, doesn't brighten
Header band goes from #25B439 to #1B8A2A. A bright green band on dark bg is blinding. The darker green still reads as brand but doesn't assault the eyes.
Hub orb gets a rim
On light bg, the near-black hub has natural contrast. On dark bg, it needs a subtle rgba(255,255,255,0.06) border to remain visible against #111.
Follow system preference
Use prefers-color-scheme by default. No manual toggle needed for v1 — respect the OS setting. If Moj uses her Mac in dark mode, the app follows.
