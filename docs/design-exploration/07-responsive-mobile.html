Map on Small Screens
The orbital map scales down naturally — orbs get smaller, orbit rings tighten. Pinch-to-zoom lets you explore. On very small screens, you could offer a list fallback, but the map should work at phone size with touch.

Mobile Map Adaptations
Touch targets: Minimum 44px tap area on each orb (Apple HIG). Smaller orbs get an invisible hit area expansion.

Pinch-to-zoom: React Flow supports this natively. Let users zoom into dense clusters.

Tap to drill: Single tap opens the pod (replaces hover). No hover states on mobile — just tap feedback (brief scale pulse).

Orbit rings: Slightly tighter on mobile to keep everything in viewport without scrolling.

Clustered small orbs: Categories cluster even tighter on mobile. Tap the cluster to fan them out.
Contact Panel on Mobile
On desktop, the panel is a right-side drawer (340px). On mobile, it becomes a full-screen sheet that slides up from the bottom with a back button. The orb map is hidden behind it — you're either in the map or in a contact list, not both.
Nav Changes
Desktop: Floating pill nav (centered, bottom-ish)
Mobile: Fixed bottom tab bar with dots. Same two tabs (Pulse / Map). Always visible, always tappable. 48px height minimum.

Full-Screen Contact Sheet
Tapping a category orb on mobile slides up a full-screen sheet. Back arrow returns to the map. Clean, focused, no split view.
