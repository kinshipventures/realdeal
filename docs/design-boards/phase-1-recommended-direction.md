# Real Deal - Phase 1 Recommended Direction

## Recommendation

Use this combination:

- Shell: Variant B from [shell-variants.md](/Users/gabrielmurray/dev/realdeal/docs/design-boards/shell-variants.md)
- Dashboard: Variant B from [dashboard-variants.md](/Users/gabrielmurray/dev/realdeal/docs/design-boards/dashboard-variants.md)
- Record Detail: Variant A from [record-detail-variants.md](/Users/gabrielmurray/dev/realdeal/docs/design-boards/record-detail-variants.md)

## Why This Combo

- it keeps green in the frame
- it gives the app one real shell
- it gives the dashboard more order without flattening it
- it gives record detail a stronger sense of identity and depth

## What The App Should Feel Like After Phase 1

- more like one designed product
- less like separate pages under one nav
- calmer at first glance
- clearer about what matters first

## Phase 1 Deliverables

1. shared shell refresh
2. dashboard layout refresh
3. record detail layout refresh

## What Waits Until Phase 2

- records list polish
- campaign overview polish
- campaign detail shell cleanup
- pods map chrome alignment

## Build Order

1. shell
2. dashboard
3. record detail
4. campaign detail

## Guardrails

- keep green as framing
- keep reading surfaces neutral
- do not add louder gradients
- do not turn every section into a card stack
- make the app feel more composed before making it more flashy
