# Real Deal - Record Detail Variants

## Goal

Make the record page feel like a real relationship workspace.

## Current Baseline

- clear two-column page
- header, signal, widgets, and timeline already exist
- good bones, but not enough composition

## Variant A - Identity Band

### Idea

Make the record header a stronger identity band, then drop into a two-column workspace.

### Layout

- top band includes breadcrumb, identity, context, and key actions
- signal message sits inside the band or just under it
- left column becomes grouped report blocks
- right column stays the active timeline surface

### Pros

- strongest page identity
- makes the contact feel important

### Cons

- easy to overbuild the header if not restrained

## Variant B - Quiet Workspace

### Idea

Keep the top lighter and let the two-column body do more of the work.

### Layout

- smaller top frame
- signal sits inline above the grid
- left column stays practical
- timeline dominates the page

### Pros

- calmer
- safer

### Cons

- less memorable
- does not solve the identity problem as well

## Recommendation

Pick Variant A.

This page should feel more like entering a relationship workspace, not just viewing a profile.

## Rules

- timeline stays the main working surface
- left side reads like a report
- signal state should feel integrated
- identity needs more weight than utility actions
