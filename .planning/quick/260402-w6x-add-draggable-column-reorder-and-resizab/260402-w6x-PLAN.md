<task type="auto">
  <name>Task 2: Add column resize via drag handles on header edges</name>
  <files>src/components/records/RecordsList.tsx</files>
  <action>
Add resize handles to column headers in RecordsList.tsx.

**State additions:**
- `columnWidths: Record<ColumnId, number>` state -- initialized from localStorage key `realdeal:contacts-column-widths`, falling back to `{}` (empty = auto width)
- `resizingCol: ColumnId | null` ref
- `resizeStartX: number` ref
- `resizeStartWidth: number` ref

**Resize handle element:**
- Inside each `<th>` (including name, all columns are resizable), add a `<div>` resize handle as the last child:
  ```
  position: absolute, right: 0, top: 0, bottom: 0, width: 4px
  cursor: col-resize, zIndex: 2
  ```
- The `<th>` needs `position: relative` added to its style.
- On hover, the handle shows a 1px vertical line: `background: var(--edge)` on hover via onMouseEnter/onMouseLeave toggling a local hover state, or simpler: use `:hover` via a CSS class `.col-resize-handle:hover { background: var(--color-text-tertiary) }`.
- Add the `.col-resize-handle` styles. Since this project uses inline styles + index.css, add a small CSS block to `src/index.css`:
  ```css
  .col-resize-handle { background: transparent; transition: background 0.15s; }
  .col-resize-handle:hover, .col-resize-handle.active { background: var(--color-text-tertiary); }
  ```

**Resize behavior:**
- `onPointerDown` on handle: `e.stopPropagation()` (prevent sort click), `e.preventDefault()`, capture pointer via `(e.target as HTMLElement).setPointerCapture(e.pointerId)`. Set `resizingCol`, `resizeStartX = e.clientX`, `resizeStartWidth` = current column width (from `columnWidths[col.id]` or measure `th.offsetWidth`).
- `onPointerMove` on handle: if `resizingCol` matches, compute `newWidth = resizeStartWidth + (e.clientX - resizeStartX)`. Clamp to minimum 60px. Update `columnWidths` state for that column.
- `onPointerUp` on handle: release pointer capture, clear `resizingCol`, persist `columnWidths` to localStorage key `realdeal:contacts-column-widths`.

**Table layout:**
- When any column has a custom width (Object.keys(columnWidths).length > 0), set `tableLayout: 'fixed'` on the `<table>` element.
- Apply `width` style to each `<th>` and corresponding `<td>`: `columnWidths[col.id] ? columnWidths[col.id] + 'px' : undefined`.
- Checkbox column stays at `width: 44`.

**Double-click to auto-size:**
- `onDoubleClick` on resize handle: remove that column from `columnWidths` (reset to auto). If no columns have custom widths, remove `table-layout: fixed`.

**Persist:**
- On resize end, write to `localStorage.setItem('realdeal:contacts-column-widths', JSON.stringify(columnWidths))`
- On mount, read from localStorage (already covered in state init)

**SavedView integration:**
- Add `columnWidths?: Record<ColumnId, number>` to `SavedView` interface
- In `saveView`: include `columnWidths`
- In `applyView`: if `view.columnWidths` exists, set it; otherwise keep current widths
- In reset (null view): reset `columnWidths` to `{}`

**Interaction with drag-reorder:**
- The resize handle's `onPointerDown` must call `e.stopPropagation()` to prevent the drag-reorder from triggering.
- The `<th>` should only be `draggable` when NOT resizing.
