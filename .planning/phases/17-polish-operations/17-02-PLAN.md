Operation order (critical - no Airtable transactions):

1. **Fetch both records** from cache or API via getContacts().
2. **Build merged field set:**
   - Start with survivor's current fields
   - Apply fieldOverrides (user's per-field picks from MergeModal)
   - Union list_ids: `[...new Set([...survivor.list_ids, ...loser.list_ids])]`
   - Union category_ids: same Set union
   - primary_list_id: use survivor's if set, else loser's
3. **Update opportunity references:** Fetch all opportunities. For each where `relationship_ids` includes loserId, PATCH to replace loserId with survivorId (if not already present). Use `request()` with PATCH to `${TABLES.opportunities}/${opp.id}`.
4. **Update project references:** Fetch all projects. For each where `relationship_ids` includes loserId, PATCH to replace loserId with survivorId. Use `request()` with PATCH to `${TABLES.projects}/${proj.id}`.
5. **Update campaign contact references:** Fetch all campaign contacts. For each where `contact_id === loserId`, PATCH to set contact_id to survivorId. Use `request()` with PATCH to `${TABLES.campaignContacts}/${cc.id}`.
6. **Move loser's interactions to survivor:** Fetch interactions for loserId. For each, PATCH the Contact linked field to point to survivorId. Use `request()` with PATCH to `${TABLES.interactions}/${interaction.id}` with `{ fields: { Contact: [survivorId] } }`.
7. **Update survivor record** with merged fields via updateContact().
8. **Write timeline event on survivor:** createInteraction with type `'merge_event'`, contact_id survivorId, notes `'Merged with [loserName] (record deleted)'`.
9. **Delete loser record:** deleteContact(loserId).
10. **Invalidate caches:** invalidateContactsCache(), invalidateInteractionsCache().

**Error handling:** Wrap steps 3-9 in try/catch. On failure, throw with descriptive message including which step failed. The UI will surface this. Do NOT silently swallow.
