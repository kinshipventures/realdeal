### 1. Fraunces visual rendering

**Test:** Open the app in a browser. Navigate to Dashboard and inspect the section headings and equity score number.
**Expected:** Text renders in Fraunces — a high-contrast, ink-trap serif with distinctive letterforms (not Playfair Display or system serif)

### 2. Dark mode visual correctness

**Test:** Set system preference to dark mode, reload the app.
**Expected:** Background goes near-black (#111111), green header deepens to #1B8A2A, text inverts to white-on-dark, orb glow adjusts
**Why human:** CSS media query effect requires visual confirmation; can't assert via grep

### 3. Orbital entrance animation feel

**Test:** Load the map view from a cleared localStorage (or incognito). Observe the pod orb entrance.
**Expected:** Hub appears first, then pod orbs fly outward from hub center with a spring-like ease (cubic-bezier(0.22, 1, 0.36, 1)) at 0.1s stagger intervals
**Why human:** Animation timing and feel require runtime observation

### 4. Mobile tab bar on a real device or emulator

**Test:** Open at viewport < 768px (DevTools device mode or real phone).
**Expected:** Bottom tab bar with Heart icon (Pulse) and Crosshair icon (Map) replacing the floating pill; contact panel slides up full-screen
**Why human:** Responsive behavior and touch targets require mobile environment

### 5. Sparkline rendering in pod cards

**Test:** Open Dashboard with real Airtable data (requires a pod with interactions in the last 30 days).
**Expected:** Each pod card shows a small inline area chart at the bottom — pod-colored line with 12% opacity fill, no sparkline for pods with no recent interactions
**Why human:** Real interaction data from Airtable required; behavior when data is absent (graceful omission) also needs visual check
