# Phase 02.1 Plan 03: Orbital Map Entrance Animation Summary

**orbit-slide animation flies pod orbs from hub center to orbital positions using CSS var translate offsets, with dashed concentric orbit rings (r=200/310/420) as a viewport-tracked SVG overlay**

- **Duration:** ~15 min

- **Completed:** 2026-03-23T18:15:00Z
- **Tasks:** 1
- **Files modified:** 3

## Accomplishments
- Replaced `orb-enter` (scale-only) with `orbit-slide` keyframe that translates from hub position using CSS custom properties `--orbit-start-x` / `--orbit-start-y`
- `ListNode` wrapper div sets CSS vars from `orbitStartX`/`orbitStartY` node data; SolidOrb's animated element inherits them
- `OrbMap.buildHomeNodes` computes per-pod hub-relative offsets and staggers delays at 0.1s increments
- Dashed orbit rings rendered as an SVG absolutely positioned over ReactFlow, `g transform` mirrors viewport pan/zoom
- Home edges updated to `rgba(0,0,0,0.06)` at `1.5px` — subtler, more orbital feel

## Task Commits

1. **Task 1: Orbit-slide entrance animation and dashed orbit ring background** - `6a03b71` (feat)

## Files Created/Modified
- `src/styles/globals.css` - orbit-slide keyframe replacing orb-enter; .orb-enter uses new animation at 0.7s cubic-bezier(0.22, 1, 0.36, 1)
- `src/components/map/OrbMap.tsx` - orbitStartX/Y calculation, viewport state tracking, SVG orbit rings overlay, updated edge styles
- `src/components/map/ListNode.tsx` - added orbitStartX/orbitStartY to ListNodeData; orbit-start wrapper div sets CSS vars

## Decisions Made
- CSS var inheritance over SolidOrb prop changes — keeps SolidOrb's interface stable, parent div sets `--orbit-start-x/y`, inner animated div inherits
- SVG g transform pattern for viewport-tracking overlay — avoids React Flow internals, purely declarative
- strokeWidth / strokeDasharray divided by `viewport.zoom` to keep ring line weight visually consistent regardless of zoom level
- Orbit rings shown in home view only — category view centers on a pod orb, rings would be misleading
