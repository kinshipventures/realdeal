# Phase 02.1 Plan 02: Orb Surface — Two-Tone Gradient + Glow Summary

**Two-tone diagonal gradient orbs with per-pod colored glow halos (hover-reactive) and optional SVG health ring arc**

## Accomplishments

- SolidOrb rewritten with linear-gradient(135deg) two-tone surface, replacing radial shimmer
- Colored glow box-shadow at 0.25 opacity at rest, 0.40 on hover — managed via ref to avoid re-renders
- SVG health ring with stroke-dasharray arc rendered when healthPercent prop provided
- POD_SHIFT_COLORS map exported for all six pod colors + hub
- MojNode upgraded to dark two-tone gradient with hub-specific glow
- ListNode and CategoryNode import and pass shiftColor from POD_SHIFT_COLORS

- `src/components/map/MojNode.tsx` — dark two-tone gradient, hub glow shadow
- `src/components/map/ListNode.tsx` — imports POD_SHIFT_COLORS, passes shiftColor
- `src/components/map/CategoryNode.tsx` — imports POD_SHIFT_COLORS, passes shiftColor

## Decisions Made

- Hover glow uses ref mutation (`orbRef.current.style.boxShadow`) rather than useState — pointer events would cause constant re-renders with state
- Health ring wraps the orb in a container div that expands by 8px — only rendered when healthPercent is provided, backward compatible
- shiftColor is optional — SolidOrb degrades gracefully to single color gradient when not passed
- POD_SHIFT_COLORS map keyed by lowercase hex from DESIGN.md; consumers do `POD_SHIFT_COLORS[color] ?? POD_SHIFT_COLORS[color.toUpperCase()]` to handle both
