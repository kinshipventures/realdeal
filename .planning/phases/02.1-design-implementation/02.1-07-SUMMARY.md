# Phase 02.1 Plan 07: Sparklines + Loading Skeleton Summary

**Inline SVG sparklines in pod cards showing 30-day interaction trend, plus skeleton shimmer replacing all dashboard loading spinners**

- **Completed:** 2026-03-23T18:00:00Z

## Accomplishments

- `Sparkline` component renders a 7-point area chart per pod — interaction counts bucketed over the last 30 days. Area fill uses pod hex color + `1F` (12% opacity). Line stroke at 1.5px. Absent when no interactions exist in the window.
- `DashboardSkeleton` full-page skeleton on initial load: green band with ring ghost + stat grid ghosts, 3 pod card ghosts, section label ghosts, 2 overdue row ghosts
- `@keyframes shimmer` + `.skeleton` class added to globals.css: left-to-right gradient sweep at 1.5s, dark mode variant, reduced-motion override
- Overdue section shows skeleton rows while contacts are loading (not a Spinner)
- Ring/stats sections show inline skeleton while interactions load (post-initial-load stagger state)

## Decisions Made

- Sparkline data uses simple interaction counts per bucket (not recency-weighted equity snapshots) — simpler, more honest representation of activity trend
- Card layout changed from row-flex to column-flex to give sparkline a full-width bottom strip — more visual impact than a cramped right-aligned sparkline
- Initial skeleton shown only when both `contactsLoading && podsLoading` (true simultaneous load) — if one resolves before the other, render real content with inline per-section skeletons
