**Airtable fetch helper** (direct REST, NOT importing from airtable.ts per RESEARCH anti-pattern):
```typescript
async function fetchAirtableTable(tableName: string): Promise<AirtableRecord[]> {
  const records: AirtableRecord[] = []
  let offset: string | undefined
  do {
    const url = `https://api.airtable.com/v0/${AIRTABLE_BASE_ID}/${encodeURIComponent(tableName)}${offset ? `?offset=${offset}` : ''}`
    const res = await fetch(url, { headers: { Authorization: `Bearer ${AIRTABLE_TOKEN}` } })
    if (!res.ok) throw new Error(`Airtable ${tableName}: ${res.status} ${await res.text()}`)
    const json = await res.json()
    records.push(...json.records)
    offset = json.offset
  } while (offset)
  console.log(`  Fetched ${records.length} records from ${tableName}`)
  return records
}
```

**ID mapping helpers:**
```typescript
const idMap = new Map<string, string>() // airtableId -> supabaseUuid

async function recordMapping(airtableId: string, tableName: string, supabaseUuid: string) {
  idMap.set(airtableId, supabaseUuid)
  await supabase.from('_migration_id_map').insert({
    airtable_id: airtableId,
    table_name: tableName,
    supabase_uuid: supabaseUuid,
    user_id: MIGRATION_USER_ID,
  })
}

function resolveId(airtableId: string): string {
  const uuid = idMap.get(airtableId)
  if (!uuid) throw new Error(`No mapping for Airtable ID: ${airtableId}`)
  return uuid
}

// Resolve array of linked record IDs, skipping any that don't resolve
function resolveIds(airtableIds: string[] | undefined): string[] {
  if (!airtableIds?.length) return []
  return airtableIds.map(id => idMap.get(id)).filter((id): id is string => !!id)
}
```

**Migration order (per D-17):**

Read the TABLES constant from airtable.ts to get the Airtable table names. The Airtable table names (from TABLES constant) are:
- Lists, Categories, Contacts, Interactions, Campaigns, CampaignContacts, Pipelines, Pipeline Stages, Opportunities, Projects, Field Config

For each table, map Airtable PascalCase field names to Supabase snake_case columns. Reference the mapper functions in airtable.ts (mapPod, mapContact, etc.) to see the exact field name mappings.

1. **Pods** (from Lists): `Name` -> `name`, `Color` -> `color`, `Owner` -> `owner`, `Is Priority` -> `is_priority`, `Cadence` -> `cadence`, `Description` -> `description`, `Capacity` -> `capacity`, `Enrichment Opt-In` -> `enrichment_opt_in`

2. **Categories**: `Name` -> `name`, `Color` -> `color`, `List` (linked record array, take first) -> resolve to `pod_id`

3. **Companies** (Contacts where Type='Company'): Split from Contacts table. `Name` -> `name`, `Industry` -> `industry`, `Stage` -> `stage`, `Ticker` -> `ticker`, `Domain` -> `domain`, `Website` -> `website`, `Location` -> `location`, `Notes` -> `notes`, `Custom Fields` -> `custom_fields`

4. **Contacts** (all Contacts records including type='Company' -- the interface expects both): All field mappings from `mapContact()`. `Company Record` (linked record) -> resolve to `company_id`. `Primary List ID` stored on contact but per D-14 moves to `contact_pods.is_primary`.

5. **contact_pods** junction: For each contact, iterate `Lists` (linked record array), insert one row per pod. Mark the pod matching `Primary List ID` as `is_primary: true`.

6. **contact_categories** junction: For each contact, iterate `Categories` (linked record array), insert one row per category.

7. **Interactions**: `Contact` (linked record, take first) -> resolve to `contact_id`, `Type` -> `type`, `Date` -> `date`, `Notes` -> `notes`, `Summary` -> `summary`, `Source` -> `source`, `Email Link` -> `email_link`, `Granola Link` -> `granola_link`, `Event Detail` -> `event_detail`, `Actor` -> `actor`

8. **Pipelines**: `Name` -> `name`, `Status` -> `status`

9. **Pipeline Stages**: `Name` -> `name`, `Pipeline` (linked record) -> resolve to `pipeline_id`, `Color` -> `color`, `Order` -> `order`

10. **Opportunities**: `Name` -> `name`, `Stage` (linked record) -> resolve to `stage_id`, `Notes` -> `notes`, `Priority` -> `priority`, `Status` -> `status`

11. **opportunity_contacts** junction: For each opportunity, iterate `Relationships` (linked record array), insert one row per contact.

12. **Campaigns**: `Name` -> `name`, `Type` -> `type`, `Deadline` -> `deadline`, `Status` -> `status`

13. **campaign_contacts**: Already a separate Airtable table (CampaignContacts). `Campaign` (linked record) -> resolve to `campaign_id`, `Contact` (linked record) -> resolve to `contact_id`, `Status` -> `status`, `Notes` -> `notes`

14. **Projects**: `Name` -> `name`, `Description` -> `description`, `Owner` -> `owner`, `Notes` -> `notes`

15. **project_contacts** junction: For each project, iterate `Relationships` (linked record array), insert one row per contact.

16. **project_opportunities** junction: For each project, iterate `Opportunities` (linked record array), insert one row per opportunity.

17. **Field Config**: `Name` -> `name`, `Airtable Field ID` -> `airtable_field_id`, `Field Type` -> `field_type`, `Scope Type` -> `scope_type`, `Scope Pod` (linked record) -> resolve to `scope_pod_id`, `Required` -> `required`, `Display Order` -> `display_order`

**Batch inserts:** Insert up to 50 records at a time via Supabase `.insert([...batch]).select()` for performance. Map over the returned rows to record ID mappings.

After all inserts, run count checks:

```typescript
const tableCounts = ['pods', 'categories', 'companies', 'contacts', 'contact_pods', 'contact_categories', 'interactions', 'pipelines', 'pipeline_stages', 'opportunities', 'opportunity_contacts', 'campaigns', 'campaign_contacts', 'projects', 'project_contacts', 'project_opportunities', 'field_config']

```

Also log the Airtable source counts for comparison.

**Error handling:** Wrap each table migration in try/catch. On error, log the failing record's Airtable ID and field data, then continue (don't abort entire migration for one bad record). Collect errors and print summary at end.
