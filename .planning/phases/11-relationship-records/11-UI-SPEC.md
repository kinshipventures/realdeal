## Interaction Contract

### Click-to-edit inline fields

- Trigger: single click on field value
- Inactive state: value text in `rgba(0,0,0,0.82)`, no border visible
- Active state: input replaces text, 1px bottom border `#25B439`, font identical to display value
- Save: `onBlur` or Enter
- Cancel: Escape (uses `useEscape` from escapeStack.ts)
- Empty value display: em dash `—` in `rgba(0,0,0,0.28)`

### Company typeahead (on Contact header and creation modal)

- Triggers after 2 characters
- Debounce: 150ms
- Results: floating dropdown, max 5 results, matching text bolded
- No results + input > 2 chars: show "+ Create '[input]' as company" option
- Dropdown dismisses on Escape (escapeStack) and on blur
- Selection: updates `company_record_id` field, displays company name as link

### Pod assignment (creation modal)

- Multi-select with chips
- Each chip uses pod color from `POD_SHIFT_COLORS` as background, white text
- At least one pod required — "Create" button disabled + tooltip "Assign at least one pod" until satisfied
- Pod assignment cannot be left empty: CRE-04 enforcement

### "+ Add field" (PodFieldsWidget)

- Appears at bottom of each pod's field card
- Click: expands inline form below the existing fields (no modal)
- Inline form fields: field name (text input), type (select: Text / Number / Select / Date / Checkbox), Required toggle
- Save button: "Add field" — brand green text button (not filled)
- While saving (Metadata API in flight): button disabled, label changes to "Adding…"
- On success: new field appears in the card immediately, inline form collapses
- On error: inline error message "Couldn't create field. Try again."

### Navigation wires

- All contact/company navigation goes to `/record/:id` (no slide-out)
- Back: `← Back` button in RecordHeader, uses `navigate(-1)` (browser history)
- ContactPanel preview cards: each row has "Open Record" text button right-aligned that calls `navigate('/record/:id')`
- SearchPalette: result selection navigates to `/record/:id`

### Conditional field rendering

- Fields that don't match record type: not rendered, no placeholder
- Pod field groups that don't match assigned pods: not rendered
- Implementation: pure conditional render (`&&`), never `display: none`

## Accessibility Contract

- All interactive elements: min 44px touch target height (Apple HIG)
- "Open Record" buttons in ContactCard: `aria-label="Open [contact.name]'s record"`
- Type/status badges: `role="status"`, `aria-label="[Type]: [Status]"`
- Required field markers: `aria-required="true"` on inputs, not just visual `*`
- Inline error messages: `role="alert"` on error containers
- Modal: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to heading
- Focus trap in CreateRecordModal (standard browser behavior via shadcn dialog)
- Escape key: dismiss modal, cancel inline edit, close typeahead — via `escapeStack.ts`
