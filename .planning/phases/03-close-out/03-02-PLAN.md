**## Importing Contacts** (references the new UI — per D-10)
Step-by-step with expected CSV format:
- CSV must have a header row
- Recognized columns: Name, Email, Phone, Company, Role, Location, Website, Notes, Specialization, Recommended By, Past Clients
- Columns not recognized are ignored (safe to include extra data)
- Dedup: contacts with matching name OR email (case-insensitive) are skipped
- All imported contacts are linked to the selected pod

**## Known Issues** (per D-12 — real bugs found during dev)
Document actual issues discovered. Examples from the codebase/development:
- Airtable API rate limit: importing large CSVs (200+ contacts) may hit the 5 requests/second limit. The import paces at 250ms between creates but bulk operations can still timeout.
- localStorage orb positions: if orb layout gets corrupted, clearing `kinshipbrain:node-positions:v2` from browser DevTools (Application > Local Storage) resets the map layout
- Equity scores assume 30-day default cadence if pod has no cadence set
- Dark mode is system-preference only — no manual toggle
- No search — finding a specific contact requires navigating through pods/categories or scrolling
- No authentication — app is accessible to anyone with the URL (acceptable for private use)
- Supabase directory in repo is stale/unused — ignore it
