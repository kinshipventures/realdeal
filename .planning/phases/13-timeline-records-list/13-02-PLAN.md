**Filter state (per D-14, LIST-01):**

```typescript
type RecencyFilter = 'any' | '7d' | '30d' | '90d' | 'never'

interface FilterState {
  search: string
  pod: string | null        // pod ID or null for all
  type: RelationshipType | null
  status: RelationshipStatus | null
  recency: RecencyFilter    // last-contact recency filter (per LIST-01)
}
```

Filter bar sits above the table — a horizontal row with:
- Text input for search (filters name, company, email)
- Pod dropdown (all pods + "All Pods" option)
- Type dropdown: All / Person / Company
- Status dropdown: All / Active / Pending / Archived
- Recency dropdown (per LIST-01): Any time / Last 7 days / Last 30 days / Last 90 days / Never contacted. Default: "Any time". Filters on `last_contacted_at` — "Never" matches null, date options match contacts whose `last_contacted_at` is within the selected number of days from `Date.now()`.

All filters are AND-combined.

**Sorting:**
- Click column header to sort. Same pattern as CategoryTable: SortCol + SortDir state, toggle on click.
- Default sort: equity desc.

**Column visibility toggle (per D-13):**
- Small "Columns" button in the filter bar (right side)
- Clicking opens a dropdown with checkboxes for each column
- State: `Set<ColumnId>` initialized from COLUMNS.filter(c => c.defaultVisible)

**Saved views (per D-15, D-16):**

localStorage key: `kinshipbrain:records-views`

interface SavedView {

- "Views" dropdown button in the filter bar (left side, before filters)

- Clicking a saved view restores its filters, columns, and sort

- Click row navigates to `/record/:id` via `useNavigate()`

**Cell rendering per column:**
- `name`: Contact name, left-aligned. If company type, show in slightly different weight.
- `company`: Raw text, muted if null ("-")
- `pod`: Primary pod name with small colored dot (pod.color). If no primary pod, show "-"
- `equity`: Numeric score + badge like CategoryTable (`EQUITY_BADGE` colors for Thriving/Steady/Cooling/Fading)

- `status`: Pill with subtle background — green for Active, amber for Pending, gray for Archived
- `last_contact`: `formatRelativeTime(contact.last_contacted_at)` or "Never"
- `cadence`: cadence_override or contact_frequency or "-"
- `location`: contact.location or "-"
- `follow_up`: `formatRelativeTime(contact.next_follow_up_date)` or "-"

- `selectedIds: Set<string>` state
- When 1+ selected, show a bulk action bar above the table (placeholder for now — Plan 03 adds the actual actions)
