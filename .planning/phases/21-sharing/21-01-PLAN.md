**1. Create migration file** `supabase/migrations/YYYYMMDD_share_links.sql` (use current date for YYYYMMDD prefix, e.g. `20260402_share_links.sql`):

```sql
CREATE TABLE share_links (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid REFERENCES auth.users NOT NULL,
  pod_id uuid REFERENCES pods(id) ON DELETE CASCADE NOT NULL,
  token text NOT NULL UNIQUE,
  excluded_contact_ids uuid[] DEFAULT '{}' NOT NULL,
  visible_columns text[] DEFAULT '{name,role,company,pod}' NOT NULL,
  expires_at timestamptz NOT NULL,
  pin_hash text,
  revoked_at timestamptz,
  created_at timestamptz DEFAULT now() NOT NULL
);

CREATE INDEX share_links_token_idx ON share_links(token);
CREATE INDEX share_links_pod_id_idx ON share_links(pod_id);

ALTER TABLE share_links ENABLE ROW LEVEL SECURITY;

-- Authenticated users can manage their own share links
CREATE POLICY share_links_authenticated ON share_links
  FOR ALL TO authenticated
  USING (true) WITH CHECK (true);

-- Anonymous users can SELECT share links by token (for public page)
CREATE POLICY share_links_public_read ON share_links
  FOR SELECT TO anon
  USING (token IS NOT NULL);

-- Anonymous users need to read contacts and contact_pods for shared views
CREATE POLICY contacts_anon_read ON contacts
  FOR SELECT TO anon
  USING (
    EXISTS (
      SELECT 1 FROM share_links sl
      JOIN contact_pods cp ON cp.pod_id = sl.pod_id AND cp.contact_id = contacts.id
      WHERE sl.revoked_at IS NULL
        AND sl.expires_at > now()
        AND NOT (contacts.id = ANY(sl.excluded_contact_ids))
    )
  );

CREATE POLICY contact_pods_anon_read ON contact_pods
  FOR SELECT TO anon
  USING (
    EXISTS (
      SELECT 1 FROM share_links sl
      WHERE sl.pod_id = contact_pods.pod_id
        AND sl.revoked_at IS NULL
        AND sl.expires_at > now()
    )
  );

CREATE POLICY pods_anon_read ON pods
  FOR SELECT TO anon
  USING (
    EXISTS (
      SELECT 1 FROM share_links sl
      WHERE sl.pod_id = pods.id
        AND sl.revoked_at IS NULL
        AND sl.expires_at > now()
    )
  );
```

**3. Create `src/lib/sharing.ts`** with these functions:

- `generateToken()` -- generates 8-char random alphanumeric token using `crypto.getRandomValues` and base36 encoding
- `hashPin(pin: string): Promise<string>` -- SHA-256 hash of pin via Web Crypto API (SubtleCrypto.digest), returns hex string
- `verifyPin(pin: string, hash: string): Promise<boolean>` -- hashes input and compares to stored hash
- `createShareLink(data: { pod_id: string; excluded_contact_ids: string[]; visible_columns: string[]; expires_in_days: 7 | 30 | 90; pin?: string }): Promise<ShareLink>` -- inserts into share_links with generated token, computed expires_at (now + days), optional pin_hash. Uses `getUserId()` pattern from airtable.ts.
- `getActiveShareLinks(podId: string): Promise<ShareLink[]>` -- fetches non-revoked, non-expired share links for a pod. Authenticated query.
- `revokeShareLink(id: string): Promise<void>` -- sets revoked_at to now() on the share link
- `getShareLink(token: string): Promise<ShareLink | null>` -- fetches share link by token. Uses the anon supabase client (no auth required). Returns null if not found.
- `getSharedContacts(shareLink: ShareLink): Promise<{ name: string; role: string | null; company: string | null; pod_name: string }[]>` -- fetches contacts in the pod, excluding excluded_contact_ids, joined with pod name. Uses anon query. Returns only the fields needed for the public view (name, role, company, pod name). Does NOT return email, phone, notes, or other private fields per D-02/SHR-02.

Import supabase from `@/integrations/supabase/client`. For anon queries, use the same `supabase` client -- the RLS policies handle access control. The `getUserId()` helper should be copied or imported from airtable.ts pattern.

The `getSharedContacts` function should:
1. Query `contact_pods` where `pod_id = shareLink.pod_id`
2. Join contacts via `contact_id`
3. Filter out `excluded_contact_ids` using `.not('contact_id', 'in', `(${ids})`)` or client-side filter
4. Query the pod name separately
5. Return only: name, role, company, pod_name

**1. Create `src/components/sharing/SharedListPage.tsx`:**

This component handles the full public share experience per D-10 through D-14.

**State machine:**
- `loading` -- fetching share link by token
- `expired` -- link not found, expired, or revoked (show branded expiration page per D-14)
- `pin_required` -- link has pin_hash, user hasn't entered PIN yet (per D-07)
- `pin_error` -- wrong PIN entered
- `ready` -- contacts loaded, render table

**Implementation:**
- Extract `token` from `useParams<{ token: string }>()`
- On mount, call `getShareLink(token)` from `src/lib/sharing.ts`
- If null, or `revoked_at` is set, or `expires_at < new Date().toISOString()` -- show expired state
- If `pin_hash` exists and PIN not yet verified -- show PIN input form
- On PIN submit, call `verifyPin(input, shareLink.pin_hash)` -- if false, show error; if true, proceed
- On valid link (no PIN or PIN verified), call `getSharedContacts(shareLink)` to get contact rows
- Filter `visible_columns` from the shareLink to determine which table columns to render (per D-13)
