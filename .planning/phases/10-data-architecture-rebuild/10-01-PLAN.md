async function airtable(path: string, options?: RequestInit) {
  const res = await fetch(`${BASE_URL}/${path}`, {
    ...options,
    headers: {
      Authorization: `Bearer ${TOKEN}`,
      'Content-Type': 'application/json',
      ...options?.headers,
    },
  })
  if (!res.ok) throw new Error(`Airtable ${res.status}: ${await res.text()}`)
  return res.json()
}

Create `src/scripts/migrateSchema.ts` following the seedLists.ts pattern (standalone script, `tsx --env-file=.env.local` runner).

**Script structure:**

1. **Helpers:**
   - `metaRequest(path, options?)` — fetch wrapper for `https://api.airtable.com/v0/meta/bases/${BASE_ID}/...` with auth header
   - `recordRequest(path, options?)` — fetch wrapper for `https://api.airtable.com/v0/${BASE_ID}/...` (same as existing seedLists pattern)
   - `getBaseSchema()` — `GET /v0/meta/bases/${BASE_ID}/tables` returns `{ tables: [{ id, name, fields: [{ name, type }] }] }`
   - `addFieldIfMissing(tableId, fieldSpec, existingFieldNames)` — idempotent field creation
   - `createTableIfMissing(name, fields, existingTableNames)` — idempotent table creation
   - `chunk<T>(arr: T[], size: number): T[][]` — batch utility
   - `delay(ms: number)` — `new Promise(r => setTimeout(r, ms))` for rate limiting

2. **Constants:**
   - `CONTACTS_TABLE_ID = 'tbll75mRMMVBGiNpj'`
   - `RATE_LIMIT_DELAY = 300` (ms between API calls)

3. **Step 1 — Add fields to Contacts table** (per D-01, D-02, D-03, D-08):
   - Fetch base schema via `getBaseSchema()`
   - Get existing field names for Contacts table
   - Add these fields if missing:
     - `{ name: 'Type', type: 'singleSelect', options: { choices: [{ name: 'Contact' }, { name: 'Company' }] } }`
     - `{ name: 'Status', type: 'singleSelect', options: { choices: [{ name: 'Active' }, { name: 'Pending' }, { name: 'Archived' }] } }`
     - `{ name: 'Industry', type: 'singleLineText' }`
     - `{ name: 'Stage', type: 'singleLineText' }`
     - `{ name: 'Ticker', type: 'singleLineText' }`
     - `{ name: 'Domain', type: 'singleLineText' }`
   - Log each field creation or skip

4. **Step 2 — Set defaults on existing contacts** (per D-02):
   - Fetch all contacts via pagination (`recordRequest` with offset handling)
   - For each contact missing `Type` field: PATCH `Type: 'Contact'`
   - For each contact missing `Status` field: PATCH `Status: 'Active'`
   - Batch PATCHes in groups of 10 with `RATE_LIMIT_DELAY` between batches
   - Log count of records updated

5. **Step 3 — Create Company records** (per D-05):
   - Re-fetch all contacts
   - Collect unique company names: `contacts.filter(c => c.fields.Company?.trim()).map(c => c.fields.Company!.trim())`
   - Deduplicate case-insensitively: build a `Map<string, string>` keyed on `name.toLowerCase()`, value = first-seen original casing
   - Create Company records in batches of 10: `{ fields: { Name: originalCasing, Type: 'Company', Status: 'Active' } }`
   - Store mapping: `Map<lowercaseName, recordId>`
   - Log count created

6. **Step 4 — Add Company Record linked field + link contacts** (per D-04):
   - Add linked field if missing: `{ name: 'Company Record', type: 'multipleRecordLinks', options: { linkedTableId: 'tbll75mRMMVBGiNpj' } }`
   - Note: Airtable will auto-create a symmetric inverse field. Log the response to capture the inverse field name.
   - For each contact that has a `Company` text value: PATCH `{ 'Company Record': [companyMap.get(company.toLowerCase())] }` where `companyMap` is from Step 3
   - Batch in groups of 10 with delay
   - Log count linked

7. **Step 5 — Create Pipelines table** (per D-06):
   - Create table if missing with fields:
     - `{ name: 'Name', type: 'singleLineText' }` (primary field)
     - `{ name: 'Pipeline Status', type: 'singleSelect', options: { choices: [{ name: 'active' }, { name: 'hidden' }] } }`
   - Note: Airtable auto-creates a `Created Time` field. The primary field MUST be the first field in the array.
   - Store returned table ID

8. **Step 6 — Create Pipeline Stages table** (per D-06):
   - Create table if missing with fields:
     - `{ name: 'Name', type: 'singleLineText' }` (primary field)
     - `{ name: 'Color', type: 'singleLineText' }`
     - `{ name: 'Order', type: 'number', options: { precision: 0 } }`
     - `{ name: 'Pipeline', type: 'multipleRecordLinks', options: { linkedTableId: pipelinesTableId } }`

9. **Step 7 — Create Opportunities table** (per D-06):
   - Create table if missing with fields:
     - `{ name: 'Name', type: 'singleLineText' }` (primary field — opportunity name)
     - `{ name: 'Stage', type: 'multipleRecordLinks', options: { linkedTableId: pipelineStagesTableId } }`
     - `{ name: 'Relationships', type: 'multipleRecordLinks', options: { linkedTableId: 'tbll75mRMMVBGiNpj' } }`
     - `{ name: 'Notes', type: 'multipleLineText' }`
     - `{ name: 'Priority', type: 'singleSelect', options: { choices: [{ name: 'high' }, { name: 'medium' }, { name: 'low' }] } }`
     - `{ name: 'Opportunity Status', type: 'singleSelect', options: { choices: [{ name: 'open' }, { name: 'won' }, { name: 'lost' }, { name: 'archived' }] } }`

10. **Step 8 — Create Projects table** (per D-07):
    - Create table if missing with fields:
      - `{ name: 'Name', type: 'singleLineText' }` (primary field)
      - `{ name: 'Description', type: 'multipleLineText' }`
      - `{ name: 'Owner', type: 'singleLineText' }`
      - `{ name: 'Relationships', type: 'multipleRecordLinks', options: { linkedTableId: 'tbll75mRMMVBGiNpj' } }`
      - `{ name: 'Opportunities', type: 'multipleRecordLinks', options: { linkedTableId: opportunitiesTableId } }`
      - `{ name: 'Notes', type: 'multipleLineText' }`

11. **Final output:**
    - Print a summary JSON object with all new table IDs:
      ```
      { pipelines: 'tblXXX', pipelineStages: 'tblXXX', opportunities: 'tblXXX', projects: 'tblXXX' }
      ```
    - This output is needed by Plan 02 to update TABLES constant in airtable.ts.

**Error handling:**

- Each step logs `[Step N/8] description...` for progress tracking
- If self-referencing linked field fails (open question from research), log the error clearly with: `"Self-referencing linked field not supported via API. Company Record field must be created manually in Airtable UI."` and continue with remaining steps.
