**Layout — centered modal overlay (per D-01):**
- Full-screen backdrop: `background: rgba(0,0,0,0.3)`, `backdrop-filter: blur(8px)`, click-to-dismiss
- Modal container: centered horizontally, positioned ~20% from top, `max-width: 480px`, `width: calc(100% - 32px)`, `border-radius: var(--panel-radius)`, `background: var(--surface-panel)`, `border: var(--surface-panel-border)`, `box-shadow: 0 16px 48px rgba(0,0,0,0.12)`
- Entrance animation: opacity 0->1 and scale 0.97->1 over 150ms ease-out (per Claude's discretion)

**Search input (per D-11):**
- Full-width input at top of modal, no visible border, `font-size: 16px` (prevents iOS zoom), `font-weight: 400`, `color: var(--color-text-primary)`, `padding: 16px 20px`
- Placeholder: "Find someone..." in `var(--color-text-tertiary)`
- Bottom border: `1px solid var(--divider)` separating input from results
- Auto-focus on mount
- Search icon (magnifying glass SVG, 16px, `var(--color-text-tertiary)`) to left of input

**Filtering logic (per D-05, D-06, D-07):**
- On mount, call `getContacts()` to get cached contacts array and `getPods()` to get pods
- Build a `Map<string, Pod>` from pods for O(1) lookup
- Filter contacts where `contact.name`, `contact.company`, or `contact.role` includes query (case-insensitive substring via `.toLowerCase().includes(query.toLowerCase())`)
- No debounce — filter on every keystroke (per D-07)
- Limit displayed results to 20 to prevent DOM bloat

**Result rows (per D-08, D-09):**
- Each row is a `<button>` with `width: 100%`, `padding: 10px 20px`, `display: flex`, `align-items: center`, `gap: 10px`
- Hover: `background: var(--tint)`
- Pod-colored dot: 8px circle, `border-radius: 50%`, color from pod's `color` field (fall back to `#718096` if null). Find pod by matching `contact.list_ids[0]` against pods map
- Name: `font-size: 13px`, `font-weight: 500`, `color: var(--color-text-primary)`, `flex: 1`, `text-align: left`, truncate with `overflow: hidden; text-overflow: ellipsis; white-space: nowrap`
- Pod name: `font-size: 11px`, `color: var(--color-text-tertiary)`, after name
- Last contact time (right-aligned): `font-size: 11px`, `color: var(--color-text-secondary)`, `flex-shrink: 0`. Format: if `last_contacted_at` is null show "Never", else compute days ago — show "Today" for 0, "1d" for 1, "2w" for 14, "3mo" for 90, etc. Use simple thresholds: <1 day="Today", <7 days=Nd, <30 days=Nw (Math.floor(days/7)), >=30 days=Nmo (Math.floor(days/30))
- On click: call `onSelectContact(contact)` (per D-09)

**Empty state (per D-10):**
- When query is non-empty and results are empty, show centered text "No one by that name" in `font-size: 13px`, `color: var(--color-text-tertiary)`, `padding: 32px 20px`

**Initial state:**
- When query is empty, show nothing (empty container, just the input)

**Keyboard navigation (per Claude's discretion):**
- Track `activeIndex` state (default -1 = none)
- Arrow Down: increment activeIndex (wrap to 0 at end)
- Arrow Up: decrement activeIndex (wrap to end at 0)
- Enter: select contact at activeIndex (if >= 0)
- Active row gets `background: var(--tint)` highlight
- Scroll active row into view with `scrollIntoView({ block: 'nearest' })`

**Escape dismiss (per D-04):**
- Use `useEscape(onClose)` from escapeStack — wrap `onClose` in `useCallback`
- Also dismiss on backdrop click (already described above)
