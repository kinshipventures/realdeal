### Phase 14: Pipelines
**Goal**: Kanban pipeline boards where opportunity cards reference relationships — no duplication
**Depends on**: Phase 13 (timeline writes pipeline events)
**Requirements**: PIPE-01, PIPE-02, PIPE-03, PIPE-04, PIPE-05, PIPE-06
**Success Criteria** (what must be TRUE):
  1. User can create a pipeline with custom named stages
  2. Opportunity cards are linked to existing relationship records — no new relationship objects created
  3. Cards can be dragged between stages, with stage changes logged to the relationship timeline
  4. Pipeline has its own fields visible only in pipeline context but queryable for reporting
  5. Navigation: open relationship from pipeline card, open pipeline from relationship record
**Plans:** TBD (needs /gsd:plan-phase)

### Phase 15: Projects
**Goal**: Projects as lightweight initiative containers distinct from pipelines
**Depends on**: Phase 14 (shares navigation patterns)
**Requirements**: PROJ-01, PROJ-02, PROJ-03
**Success Criteria** (what must be TRUE):
  1. User can create a project (event, outreach campaign, initiative) and attach contacts + opportunities
  2. Each project has a notes/updates feed
  3. Simple overview page — no dashboards or analytics yet (V2)
**Plans:** TBD (needs /gsd:plan-phase)

### Phase 16: Basic Enrichment
**Goal**: Minimal AI-free enrichment layer that respects user control
**Depends on**: Phase 11 (record fields), Phase 13 (timeline)
**Requirements**: ENRICH-01, ENRICH-02, ENRICH-03
**Success Criteria** (what must be TRUE):
  1. Email metadata (last interaction date + subject preview) shown on contact records
  2. Web search enrichment writes results to timeline with before/after values logged
  3. Enrichment is opt-in per contact or per pod — never runs automatically on all contacts
**Plans:** TBD (needs /gsd:plan-phase)

### Phase 17: Reporting + Exports
**Goal**: Essential reports that validate the entire data model, plus exports and share links
**Depends on**: Phase 14 (pipeline data), Phase 12 (pod data)
**Requirements**: RPT-01, RPT-02, RPT-03, RPT-04, RPT-05, SHARE-01, SHARE-02
**Success Criteria** (what must be TRUE):
  1. Pod distribution report shows contact counts per pod/subpod
  2. Pipeline conversion/velocity report shows stage progression and time-in-stage
  3. Engagement report shows touchpoint activity over time
  4. Any report or filtered list can be exported as CSV
  5. Read-only share links can be generated for curated lists
**Plans:** TBD (needs /gsd:plan-phase)

### Phase 18: Nurturing + Dashboard
**Goal**: Dashboard becomes the primary operating surface with maintenance hub and follow-up signals
**Depends on**: Phase 14 (pipeline data for dashboard widgets), Phase 12 (pod cadence)
**Requirements**: NURT-01, NURT-02, NURT-03, NURT-04, NURT-05
**Success Criteria** (what must be TRUE):
  1. Single maintenance feed surfaces stale relationships, upcoming milestones, and missing required info
  2. Follow-ups are dashboard-led signals (not tasks) — feel instructional, not assigned
  3. Pod cadence drives default follow-up behavior; record-level overrides are explicit and visible
  4. Quick actions available: add note, create follow-up, snooze, mark done
**Plans:** TBD (needs /gsd:plan-phase)

### Phase 19: Sharing + Polish
**Goal**: Share links for curated lists and final guardrail enforcement
**Depends on**: Phase 17 (exports/reports)
**Requirements**: SHARE-01, SHARE-02, XCUT-05, XCUT-06, XCUT-07
**Success Criteria** (what must be TRUE):
  1. Read-only share links work for LP lists, referrals, city guides
  2. No duplicate records exist anywhere in the system
  3. No silent overwrites — all changes logged with before/after values
  4. All data mutations are traceable and explainable
**Plans:** TBD (needs /gsd:plan-phase)
